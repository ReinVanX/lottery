# 年会抽奖工具 — 设计规格说明书

**日期**: 2026-07-06
**状态**: 已确认
**交付物**: 单个静态 HTML 文件（`lottery.html`），双击浏览器打开即可使用

---

## 1. 项目概述

为一个大型年会活动提供大屏幕抽奖工具。纯静态 HTML 文件，双击即用，零依赖后端，所有数据持久化于浏览器 localStorage。视觉风格为炫酷科技风（暗色背景、霓虹光效、粒子动画），操作流程支持逐个揭晓中奖者，不在场可作废重抽。最终提供中奖结果汇总和 CSV 导出供服务台核验。

---

## 2. 技术方案

**方案 B：DOM UI + Canvas 动画层**

- **DOM 层**：HTML/CSS 负责所有交互界面（配置表单、按钮、结果列表），响应式适配大屏
- **Canvas 层**：全屏覆盖的 `<canvas>` 元素负责所有动画（名字滚动、粒子效果、光效），仅在抽奖动画期间激活
- **零外部运行时依赖**：所有库（Papa Parse CSV 解析、SheetJS Excel 解析）内联到 HTML 中；Web Audio API 合成音效，不依赖音频文件
- **离线可用**：整个文件自包含，年会现场无网络也能正常运行

---

## 3. 页面架构

应用为状态机驱动的单页应用，包含 4 个视图，同一时刻仅显示一个：

```
┌────────────────────────────────────────────────────┐
│                   Lottery App                       │
│                                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
│  │① 配置页   │→│② 名单导入 │→│③ 抽奖主页（核心） │  │
│  │ 奖项设置  │  │ 上传预览  │  │ 含 Canvas 动画层  │  │
│  └──────────┘  └──────────┘  └──────────────────┘  │
│                                     │               │
│                               ┌─────▼──────────┐    │
│                               │④ 结果汇总页    │    │
│                               │ 全部中奖名单    │    │
│                               └────────────────┘    │
└────────────────────────────────────────────────────┘
```

### 3.1 ① 配置页

自由表格编辑器，可添加/编辑/删除奖项：

| 奖项名称 | 中奖人数 | 操作 |
|---------|---------|------|
| 三等奖   | 10      | 🗑️   |
| 二等奖   | 5       | 🗑️   |
| 一等奖   | 2       | 🗑️   |
| 特等奖   | 1       | 🗑️   |
| [+ 添加奖项] |         |      |

- 奖项名称：自由文本输入
- 中奖人数：数字输入，≥1
- 底部实时显示「总计将抽出 N 人」
- 数据自动保存到 `localStorage`（key: `lottery_config`）
- 按钮：「保存并进入下一步 →」

### 3.2 ② 名单导入页

- **支持格式**：CSV（`.csv`）、Excel（`.xlsx`）
- **上传方式**：拖拽文件到区域或点击选择文件
- **姓名列识别**：自动识别首列或包含「姓名」「名字」「name」关键词的列名；也允许用户手动选择列
- **预览区**：解析后展示名单，分页显示（每页 50 条），顶部显示总人数
- **确认按钮**：「确认名单，共 {n} 人 →」
- 名单存入 `localStorage`（key: `lottery_candidates`、`lottery_original`）

### 3.3 ③ 抽奖主页（核心）

**组件**：
- **Canvas 动画层**：全屏覆盖，抽奖时激活
- **信息栏**（顶部）：当前奖项名称、当前进度「第 x/n 位」
- **待机画面**：抽奖间隙显示粒子背景 + 剩余候选人数 + 下一奖项预览 + 「开始抽取下一位」按钮
- **结果确认栏**（底部，揭晓后出现）：「✅ 确认中奖」「❌ 作废重抽」
- **静音按钮**：始终可见

**流程**：每轮抽一位，逐个揭晓。多人奖项循环抽取直到该奖项名额用尽，然后进入下一奖项或跳转汇总页。

### 3.4 ④ 结果汇总页

- 按奖项等级从高到低排列
- 每张中奖名字卡片金色边框 + 微光效果
- 背景保持流动粒子
- 显示共计抽出人数
- **导出 CSV**：文件名 `lottery-results-YYYY-MM-DD.csv`，字段为 `奖项,姓名,中奖时间`
- **打印**：`window.print()`，CSS `@media print` 隐藏动画和按钮，仅保留文字结果

## 4. 动画设计（核心亮点）

### 4.1 视觉基调

- 背景色：深空黑 `#0a0a0f`
- 主色调：青蓝霓虹 `#00f0ff` + 紫罗兰 `#a855f7` 渐变
- 高亮色：金色 `#fbbf24`（用于中奖者名字）
- 全局点缀：流动粒子、扫描线、微弱色差效果
- 字体：系统等宽或无衬线，大号粗体

### 4.2 单人抽奖动画（4 阶段）

**阶段 1 — 蓄势**（2-3秒）
- Canvas 全屏覆盖，背景粒子从四周向中心汇聚
- 屏幕中央显示：「{奖项名} · 第 {x}/{n} 位」
- 文字呼吸式发光脉冲
- 底部显示剩余候选人数

**阶段 2 — 滚动**（3-5秒）
- 候选名字以竖直 slot-machine 风格高速翻转滚动
- 每个名字卡片霓虹边框 + 拖尾光效
- 背景粒子加速流动，营造速度感
- 速度曲线：慢 → 快 → 峰值
- Canvas 上的名字即为随机打乱的候选列表，滚动本身就是随机过程的视觉化

**阶段 3 — 揭晓**（2-3秒）
- 名字逐个减速，最终名字"咔"一声停住
- 屏幕剧烈闪白一下
- 中奖名字从霓虹蓝渐变为金色，字号放大
- 粒子在名字周围爆炸扩散
- 顶部显示：「🎉 {奖项名} 中奖者」

**阶段 4 — 确认**（等待操作人点击）
- 中奖名字大字定格屏幕中央，金色发光
- 两个按钮：
  - ✅ **确认中奖**：将该人从候选池移除，记录到 `lottery_winners`
  - ❌ **作废重抽**：将该人从候选池移除（不再放回），该名额重新抽取；播放"划掉消散"动效后回到阶段 1

### 4.3 待机/过渡画面

- 流动粒子背景
- 当前奖项进度：「{奖项名} 已抽出 {x}/{n} 人」
- 已中奖者名字小字排列在下方
- 剩余候选人数
- 「开始抽取下一位」按钮

### 4.4 音效（Web Audio API 合成，可静音）

| 时机 | 音效描述 |
|------|---------|
| 滚动中 | 低频嗡鸣 + 快速滴答 |
| 减速中 | 滴答逐渐变慢 |
| 揭晓瞬间 | 高音上扬 → 回落 + 低频重击 |
| 确认后 | 短促明亮琶音 |
| 作废重抽 | 低沉"嗡——" + 消散效果 |

界面始终显示静音切换按钮。

## 5. 数据管理

### 5.1 localStorage 键设计

| Key | 内容 | 说明 |
|-----|------|------|
| `lottery_config` | 奖项配置 JSON | `[{name, count}]` |
| `lottery_candidates` | 当前候选名单 | 剩余未中奖者 `string[]` |
| `lottery_original` | 原始导入名单 | `string[]`，用于回退/对比 |
| `lottery_winners` | 已确认中奖记录 | `[{name, prize, time}]` |
| `lottery_state` | 当前状态 | `{view, currentPrizeIndex, currentSlotIndex}` |

### 5.2 容错设计

- **页面刷新**：从 localStorage 恢复全部状态，回到刷新前位置
- **浏览器崩溃**：同上，所有关键数据持久化
- **误关闭标签页**：同上
- **全部重置**：设置面板提供「重置全部数据」按钮，需二次确认

## 6. 边界情况 & 错误处理

| 场景 | 处理方式 |
|------|---------|
| 名单人数少于奖项总人数 | 导入时警告，显示差额，阻止进入抽奖流程 |
| 抽到最后一人 | 不展示滚动动画，直接揭晓（仅剩一人无需随机） |
| 所有奖项抽完 | 自动跳转结果汇总页 |
| 上传了空文件 | 提示"未识别到任何姓名" |
| 上传了非 CSV/Excel 文件 | 提示"仅支持 CSV 和 Excel 格式" |
| 作废重抽后候选池为空 | 提示"候选池已无可用人员"，该奖项终止 |
| 关闭浏览器再打开 | 恢复上次进度，提示是否继续或重新开始 |

## 7. 测试要点

1. **名单解析**：CSV UTF-8、CSV GBK、Excel `.xlsx`、含表头/无表头、单列多列
2. **配置编辑**：添加/删除/修改奖项，边界值（人数=1，人数>名单总数）
3. **抽奖流程**：正常抽完一个多人奖项 → 下一个奖项 → 汇总页
4. **作废重抽**：作废后名额数不变，被作废者不再出现
5. **持久化恢复**：抽奖中途刷新 → 恢复状态 → 继续
6. **导出 CSV**：文件内容正确，中文不乱码（UTF-8 BOM）
7. **打印**：打印预览隐藏动画和按钮
8. **大屏适配**：1920×1080 和 3840×2160（4K）分辨率下正常显示
9. **浏览器兼容**：Chrome、Edge（活动现场最可能使用的浏览器）
